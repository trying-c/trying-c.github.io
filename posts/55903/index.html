Notion写作同步更新Hexo博客

发表于2025-08-29|Hexo
|总字数:3k|阅读时长:13分钟|浏览量:

自建站以来，我发现Hexo博客更新文章的操作相对繁琐，不仅要处理 Markdown 格式转换，还要手动上传图片，调整链接，设置 Front-matter……这大量的重复性工作不仅耗时，还容易出错，这些都十分容易影响个人写作的积极性。

所以，今天我终于下定决心解决这个问题——搭建一套基于 Notion API、Node.js 脚本和 GitHub Actions 的自动化工作流。即在 Notion 中完成写作，然后通过定时程序把新发布的文章自动同步更新到 Hexo 博客，包括文章内容、图片、标签、分类和发布日期等，从而把精力集中在内容创作本身！

✨ 为什么选择 Notion + Hexo + GitHub Actions？

  • Notion：强大的内容创作中枢
  • Hexo：轻量高效的静态博客
  • GitHub Actions：免费强大的自动化引擎
    这将是你的最佳组合，完美结合了 Notion 的灵活性和 Hexo 的高效性，并由 GitHub Actions 作为中枢，将两者紧密连接起来。

🛠️ 准备工作：在开始之前

在正式开始配置之前，请确保你已完成以下步骤：

  1. 一个运行正常的 Hexo 博客 ：并且其源代码已托管在 GitHub 上。

  2. 一个 Notion 账号 ：你将在其中创建博客内容数据库。

  3. 一个 GitHub 账号 ：用于托管 Hexo 博客源代码和运行 GitHub Actions。

2. 创建 Notion 集成 (API Key)

为了让 Node.js 脚本能访问你的 Notion 数据库，你需要创建一个 Notion 集成。

  1. 访问 Notion Developers页面。

  2. 点击 New integration (新集成)。

  3. 填写集成名称（例如：Hexo Sync Integration）。

  4. 选择关联的工作区。

  5. 点击 Submit创建。

  6. 创建成功后，你将看到一个 Internal Integration Token (内部集成令牌)，将其复制并妥善保存，这就是我们的 NOTION_TOKEN。这个令牌只显示一次，务必保存好。

3. 分享数据库给集成

最后一步，将你刚刚创建的博客数据库分享给这个集成：

  1. 回到你的 Notion 博客数据库页面。

  2. 点击右上角的Share(共享) 按钮。

  3. 点击 Invite(邀请)。

  4. 在搜索框中找到你刚刚创建的集成名称，并选择它。确保赋予它Can edit(可编辑) 权限（虽然我们通常只读取，但为了将来可能的功能扩展，直接给edit权限最省心）。

4. 获取数据库 ID

Notion 数据库的 ID 隐藏在数据库页面的 URL 中。

打开你的博客数据库页面，观察 URL，它通常长这样：

https://www.notion.so//?v=

database_id就是 URL 中斜杠/后面，问号 ?前面的那串约 32 位的字符。将其复制并妥善保存，这就是我们的 NOTION_DATABASE_ID。

至此，Notion 端的配置就完成了。你已经有了 NOTION_TOKEN和 NOTION_DATABASE_ID。

🚀 第四步：部署与测试

1. 提交并推送所有更改：将 sync.js 和 .github/workflows/deploy.yml 以及你对 _config.yml 的更改提交到你的 GitHub 仓库的 main 分支。

2. 查看 GitHub Actions 运行状态：

  • 在你的 GitHub 仓库页面，点击 Actions 选项卡。
  • 你会看到一个名为 Deploy Hexo to GitHub Pages 的工作流正在运行或等待运行。
  • 点击它，你可以查看详细的运行日志，确保每个步骤都成功执行。

3. 手动触发（可选）：

  • 如果你想立即测试，除了 push 触发外，也可以在 Actions 选项卡中，选择左侧的工作流 Deploy Hexo to GitHub Pages，然后点击右侧的 Run workflow 按钮，选择 main 分支并点击 Run workflow 手动触发一次。

4. 验证博客：

  • 一旦 GitHub Actions 运行成功，你的 Hexo 博客（部署在 GitHub Pages）就会自动更新。
  • 访问你的博客地址，检查 Notion 中 Published 状态为 true 的文章是否已正确同步。
